HTTP auf einen Blick

Viele Webentwickler machen sich bei der Benutzung von HTTP keine Gedanken über das eigentliche Protokoll. Diese Seite soll dem einfachen Verständnis von HTTP 1.1 dienen und helfen, das World Wide Web für Webanwendungen besser nutzbar zu machen.

Diese "HTTP Erklärung" ist lediglich eine vereinfachte, unvollständige Zusammenfassung der vollständigen Spezifikation in allgemeinverständlicher Sprache.

Gegeben sei

Eine URL, URI oder IRI, genannt Resource, die als Schema http hat.

Beispiel

http://example.com
http://www.google.de/?q=http
http://example.org/à-propos/mère

mit den folgenden Möglichkeiten

  • GET

    Mit der GET Anfrage kann der Inhalt einer Resource erfragt werden.

  • PUT

    Mit der PUT Anfrage kann der Inhalt einer Resource ersetzt werden.

  • POST

    Mit der POST Anfrage kann eine neue Resource erstellt werden.

  • DELETE

    Mit der DELETE Anfrage kann eine Resource entfernt werden.

  • HEAD

    Mit der HEAD Anfrage kann der Zustand einer Resource erfragt werden. Diese Anfrage ist Identisch zu der GET Anfrage, liefert jedoch keinen Inhalt.

  • OPTIONS

    Mit der OPTIONS Anfrage können die Möglichkeiten der Resource erfragt werden.

mit den folgenden Antworten

  • 200 - Erfolgreich

    Die Antworten 200 bis 206 signalisieren, dass die Anfrage erfolgreich war.

  • 300 - Weiterleitung

    Die Antworten 300 bis 307 signalisieren, dass die Anfrage an eine andere Resource gesendet werden soll.

  • 400 - Falsche Anfrage

    Die Antworten 400 bis 417 signalisieren, dass die Anfrage falsch gestellt wurde.

  • 500 - Serverfehler

    Die Antworten 500 bis 505 signalisieren, dass die Anfrage aufgrund eines Serverfehlers nicht beantwortet werden konnte.

Und jetzt?

Jetzt da du die mehr oder weniger wichtigen Punkte über HTTP 1.1 weißt, kannst du tiefer eintauchen.

  • Entwickle echte RESTful Architekturen
  • Denke über die Verwendung von HTTP-Methoden in deinen vergangen Projekten nach und wie du den Einsatz in Zukunft verbesser kannst
  • Erfahre mehr direkt aus der HTTP 1.1 Spezifikation and lerne etwas über headers und all die Status-Codes und Methoden, die hier nicht erwähnt wurden.
